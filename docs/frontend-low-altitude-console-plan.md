# 低空运行态势控制台升级规划

本文档定义 SkyGrid 控制台从“能看”升级为“低空运行态势台”的页面边界和最小落地范围。

## 设计目标

控制台不做营销式首页，而是直接服务低空运行管理人员：

- 快速判断今日低空任务是否拥堵。
- 快速发现待审批、硬冲突、风险冲突和消息积压。
- 能从航线态势进入审批、冲突消解和一致性监控。
- 能把 SkyGrid 的服务治理能力可视化展示出来。

## 页面一：低空态势总览

核心指标：

| 指标 | 含义 |
| --- | --- |
| 今日任务数 | 当日预约或航线任务总数 |
| 待审批任务 | 仍处于待审批状态的申请 |
| 硬冲突数量 | 同一网格、高度层、时间片重叠的冲突 |
| 风险冲突数量 | 风险网格、相邻风险或禁飞区相关提示 |
| 资源占用率 | 当前时间窗口内可用资源被占用比例 |
| 消息积压量 | Outbox 或 RabbitMQ 待处理消息 |
| 服务健康状态 | Gateway、booking、resource、notify 等服务状态 |

中间区域为 2.5D 网格地图：

- Grid 占用状态。
- 禁飞区。
- 风险区。
- 航线。
- 冲突点。
- 高度层切换。
- 时间轴滑块。

## 页面二：预约审批工作台

核心内容：

- 预约申请列表。
- 申请单详情。
- 航线预览。
- TimeSlot 占用序列。
- 冲突检测结果。
- 审批通过 / 驳回。
- 审批流转记录。

审批员在该页应能回答三个问题：

```text
这条航线占用了哪些 Grid？
这些 Grid 在对应高度层和时间片是否冲突？
审批通过后会写入多少条 resource_occupancy？
```

## 页面三：冲突消解中心

核心内容：

- 硬冲突列表。
- 风险冲突列表。
- 冲突原因。
- 冲突双方任务。
- 建议处理方式。

建议文案示例：

```text
冲突原因：Grid=G-08-12，Level=L120，TimeSlot=10:15-10:20 重叠
处理建议：调整高度层至 L150 或延后 10 分钟
```

冲突消解中心不一定在 v0.1.0 实现自动改航线，但应能把冲突原因解释清楚。

## 页面四：一致性与监控面板

核心指标：

| 指标 | 来源 |
| --- | --- |
| Outbox 待投递数量 | `GET /api/bookings/outbox/summary` |
| Outbox 失败数量 | `GET /api/bookings/outbox/summary` |
| RabbitMQ 队列状态 | RabbitMQ 管理端或后端聚合接口 |
| 通知成功率 | `notify_record` 汇总 |
| 重试次数 | `outbox_message.retry_count` |
| 补偿任务次数 | Outbox dispatch 记录或日志 |
| 限流触发次数 | Resilience4j / Micrometer |
| 熔断状态 | Actuator / Resilience4j |

这个页面用于证明 SkyGrid 不只是预约表单，而是具备消息一致性和服务治理可观测性。
